# Brief — Sales Overview (EXAMPLE)

> Filled example of `power-bi/01-brief/brief-template.md`. Paired with
> [synthetic-data — demo financials](synthetic-data-demo-financials.md), which generates the data
> this report runs on. Copy to `power-bi/projects/<name>/brief.md` and adapt.

Last updated: 2026-05-24

## 1. Audience & decision

- **Primary audience:** regional sales managers + the VP of Sales (mixed).
- **Decision this report supports:** the weekly sales review — which segments/products are behind plan and where to intervene.
- **Decision cadence:** weekly.
- **Consumption channel:** Power BI Service (desktop) + occasional printed PDF for the Monday meeting.

## 2. Source data & model

- **Semantic model:** thick PBIP (`financials` fact + simple dimensions).
- **Source systems:** demo/synthetic — generated by the synthetic-data blueprint (no live source).
- **Refresh cadence:** on-demand (it's a demo model).
- **Volume:** ~5,000 rows in `financials`.
- **Sensitive data:** no — fully synthetic.

## 3. KPIs (2–5 max)

20% change test applied — each drives a different action.

| KPI | Measure name | Target source | Format |
|---|---|---|---|
| Total Sales | `[Total Sales]` | Median month (`[Sales Target]`) | $#,##0 |
| Profit % | `[Profit %]` | 12% threshold | 0.0% |
| Discount % | `[Discount %]` | ≤ 10% guardrail | 0.0% |
| # Products to 80% | `[# Products to 80%]` | — (Pareto context) | #,##0 |

## 4. Pages & layout

| Page | Purpose | Key visuals | Layout pattern |
|---|---|---|---|
| Overview | exec summary | KPI row + monthly trend + sales-by-segment bar + detail table | 3-30-300 |
| Variance | beat/miss vs target | actual-vs-target variance chart (per month) | full-width |
| Pareto | concentration | Pareto of sales by product (80% line) | 2-up |

Three pages — do not invent extras.

## 5. Branding & style

- **Theme:** sqlbi default (or `projects/themes/blupulse` from the theme library).
- **Primary brand colors:** theme tokens, not per-visual hex.
- **Logo:** none for the demo.
- **Font:** Segoe UI.

## 6. Constraints & non-goals

- **Performance:** sub-second visuals (small model — trivial).
- **Accessibility:** WCAG AA; pair sentiment color with arrows/icons.
- **Page size:** 1280×720.
- **Don't build:** drill-through pages, RLS, mobile layout (out of scope for the demo).
- **Deferred:** YoY comparison (no prior-year data generated yet).
